# 校园零食平台二开方案

## 1. 项目目标

将普通点餐/外卖小程序改造成校园零食售卖平台，覆盖学生购买、校园代理点接单、配送到寝室、自提核销和代理佣金统计。

## 2. 业务角色

### 学生用户

- 浏览零食商品
- 加入购物车
- 下单支付
- 选择配送到寝室或代理点自提
- 查看订单状态

### 校园代理点

可理解为楼长、宿舍代理、校园小仓或合作店铺。

- 查看分配给自己的订单
- 确认接单
- 配送到寝室
- 自提订单核销
- 查看销售额和佣金

### 平台管理员

- 管理商品、分类、库存
- 管理校区、宿舍楼、代理点
- 管理订单和配送范围
- 设置佣金比例
- 查看经营数据

## 3. 核心改造思路

参考 `woniudiancang/bee` 的点餐/外卖逻辑，不直接照搬业务名称，而是做校园化映射：

| 原点餐/外卖概念 | 校园零食平台概念 |
|---|---|
| 门店 | 校园代理点/校内小仓 |
| 桌号 | 宿舍楼/寝室号 |
| 外卖配送 | 送到寝室 |
| 到店自提 | 代理点自提 |
| 店员核销 | 代理点核销 |
| 服务范围 | 校区/宿舍楼配送范围 |

## 4. 推荐订单流程

```text
学生选择商品
→ 加入购物车
→ 选择配送方式：送寝 / 自提
→ 填写校区、宿舍楼、寝室号
→ 系统匹配代理点
→ 创建订单
→ 支付成功
→ 代理点接单
→ 配送/自提核销
→ 订单完成
→ 计算代理佣金
```

## 5. MVP 版本优先级

### 第一阶段：能卖货

- 商品列表
- 商品分类
- 购物车
- 下单
- 寝室地址
- 订单列表

### 第二阶段：能配送

- 校区管理
- 宿舍楼管理
- 代理点管理
- 订单分配
- 配送状态流转

### 第三阶段：能做平台

- 代理佣金
- 数据统计
- 库存预警
- 优惠券
- 秒杀活动
- 会员体系

## 6. 注意事项

- 校园配送地址必须结构化，不要只让用户填一行文本，否则后面分单会乱成一锅粥。
- 代理点权限必须独立，不要让代理点看到全平台订单。
- 库存要区分平台总仓库存和代理点库存。
- 配送到寝室建议先做手动接单，后续再做自动派单。
