# 校园零食售卖平台产品方案终版

## 1. 产品定位

校园零食售卖平台是面向高校学生的宿舍场景即时零售系统，核心解决学生夜间、课间、宿舍内的零食饮料和生活用品即时购买需求。

平台由三端组成：

- 学生端小程序：下单、支付前流程、地址、订单。
- 校园代理端：接单、配送、自提、库存、佣金。
- 管理后台：商品、订单、代理点、校区、数据运营。

产品不是普通商城，而是围绕校园宿舍高频即时消费设计。

## 2. 核心价值

### 对学生

- 不出宿舍即可购买零食、饮料、泡面、日用品。
- 支持送到寝室和代理点自提。
- 晚间和考试周需求更集中，购买路径更短。

### 对校园代理

- 低门槛接单配送，赚取佣金。
- 每栋楼可形成小型履约点。
- 平台提供商品、订单和结算工具。

### 对平台运营方

- 可复制到多个校区。
- 可沉淀学生消费数据。
- 可通过商品差价、配送费、代理供货、广告位、活动运营盈利。

## 4. 业务模式

推荐采用混合模式：

```text
平台统一商品与价格
代理点承担履约
部分爆品平台统一供货
部分商品可由代理点补充库存
```

盈利方式：

- 商品毛利
- 配送费
- 代理点供货价差
- 校园广告位
- 活动资源位
- 会员权益

## 5. 核心流程

### 学生下单流程

```text
进入小程序
→ 选择校区 / 宿舍楼
→ 浏览商品
→ 加入购物车
→ 选择送到寝室或自提
→ 确认订单
→ 提交订单
→ 代理点接单
→ 配送或自提
→ 订单完成
```

### 代理履约流程

```text
收到订单
→ 查看商品和地址
→ 接单
→ 备货
→ 配送到寝室 / 等待自提
→ 完成订单
→ 生成佣金记录
```

### 后台管理流程

```text
配置校区和楼栋
→ 配置代理点
→ 上架商品
→ 查看订单
→ 处理订单状态
→ 查看销售数据
→ 结算代理佣金
```

## 8. 配送和费用设计

建议规则：

```text
自提：0 元
送寝：2 元
满 25 元免配送费
夜间服务费：23:00 后加 1 元，可后续开启
```

配送状态：

```text
待接单
已接单
配送中
已完成
已取消
```

自提状态：

```text
待接单
待自提
已完成
```

## 9. 佣金设计


## 14. 产品结论

校园零食平台的核心不是“做一个商城”，而是做一个贴近宿舍即时消费的履约网络。

真正的关键在于：

```text
学生端购买足够快
代理端履约足够简单
后台运营足够清晰
校区复制足够低成本
```

当前仓库已经完成可二开的 MVP 骨架，下一步应重点补微信登录、支付、真实用户体系和生产部署。
